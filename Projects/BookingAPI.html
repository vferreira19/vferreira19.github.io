ls ./features/ everything you need to build a booking system
//
multi-business management
Manage unlimited businesses under a single Developer account. Each business is fully isolated — bookings, services and users are scoped automatically.
//
smart booking validation
Built-in overlap detection, business hours checking and service availability validation. Bad data never reaches your database.
//
role-based access control
Three role levels — Developer, BusinessOwner and User — each with carefully scoped permissions. No overprivileged tokens.
//
business hours engine
Set opening and closing times per day of the week. Mark days as closed. The API enforces these rules on every booking attempt automatically.
//
service configuration
Define services with custom names, pricing and durations. Duration is used by the booking engine to calculate slot availability and prevent overlap.
//
booking lifecycle
Full booking lifecycle support — create, confirm and cancel. Status tracking built in. Business Owners can confirm or cancel. Customers can cancel their own.

cat quickstart.md get your first booking in minutes
  • 01
    subscribe on RapidAPI
    Choose a plan and subscribe. Contact us via the RapidAPI discussion board with your email to get your Developer account created.
  • 02
    login and get your token
    Call POST /api/Account/Login with your credentials. You'll receive a JWT token valid for 8 hours.
  • 03
    create your business
    Call POST /api/Business to create your first business. Pass your token in the Authorization header.
  • 04
    register a business owner
    Call POST /api/Account/RegisterOwner to create a Business Owner account for your client.
  • 05
    set up services and hours
    Business Owner adds services via POST /api/Services and sets opening hours via POST /api/Business/hours.
  • 06
    start taking bookings
    Register customers and let them book via POST /api/Booking. The API handles all validation automatically.
example — create a booking
// POST /api/Booking
// Authorization: Bearer <token>

{
  "serviceID": 1,
  "businessID": 1,
  "date": "2026-04-01T10:00:00"
}

// Response — 201 Created
{
  "bookingID": 42,
  "status": "Pending",
  "date": "2026-04-01T10:00:00",
  "serviceID": 1,
  "businessID": 1
}
automatic validation includes:
  • ✓ date is in the future
  • ✓ service exists and is available
  • ✓ business is open at requested time
  • ✓ no overlapping bookings
